# 테스트 시나리오

---

## 회원 애그리거트

### Nickname

- [ ] 유효한 형식으로 닉네임으로 생성할 수 있다
- [ ] 유효하지 않은 형식으로 닉네임을 생성할 수 없다
- [ ] 동등성을 보장한다

### Email

- [ ] 유효한 이메일 형식으로 생성할 수 있다
- [ ] 유효하지 않은 형식으로 생성할 수 없다
- [ ] 동등성을 보장한다

### Password

- [ ] 유효한 비밀번호로 생성할 수 있다
- [ ] 유효하지 않은 형식으로 생성할 수 없다
- [ ] 생성된 비밀번호는 해싱된 값을 저장한다

### Receiver

- [ ] 유효한 이름과 전화번호로 생성할 수 있다
- [ ] 유효하지 않은 형식의 이름이나 전화번호로 생성할 수 없다
- [ ] 동등성을 보장한다

### Member

#### 회원 생성
- [ ] 유효한 정보로 회원을 생성하면 ACTIVE 상태가 된다
- [ ] 회원을 생성하면 가입 시간이 기록된다
- [ ] 회원을 생성하면 비밀번호가 해싱되어 저장된다

#### 회원 탈퇴
- [ ] `ACTIVE` 회원은 탈퇴할 수 있다
- [ ] 탈퇴하면 탈퇴 시간이 기록된다
- [ ] 이미 탈퇴한 회원은 다시 탈퇴할 수 없다

#### 주문 가능 여부 확인
- [ ] `ACTIVE` 회원은 주문 가능 여부 검증을 통과한다
- [ ] `WITHDRAWN` 회원은 주문 가능 여부 검증을 통과하지 못한다

### ShippingAddress

#### 배송지 추가
- [ ] 배송지를 추가할 수 있다
- [ ] 처음 추가한 배송지는 자동으로 기본 배송지가 된다
- [ ] 두 번째 추가한 배송지는 기본 배송지가 되지 않는다
- [ ] `WITHDRAWN` 회원은 배송지를 추가할 수 없다

#### 기본 배송지 설정
- [ ] 기본 배송지를 변경할 수 있다
- [ ] 기본 배송지를 변경하면 기존 기본 배송지는 해제된다
- [ ] 이미 기본 배송지인 배송지를 다시 기본으로 설정해도 정상 처리된다
- [ ] `WITHDRAWN` 회원은 기본 배송지를 설정할 수 없다
- [ ] 본인 배송지가 아니면 기본 배송지로 설정할 수 없다

#### 배송지 수정
- [ ] 배송지를 수정할 수 있다
- [ ] `WITHDRAWN` 회원은 배송지를 수정할 수 없다
- [ ] 본인 배송지가 아니면 수정할 수 없다

#### 배송지 삭제
- [ ] 배송지를 삭제할 수 있다
- [ ] `WITHDRAWN` 회원은 배송지를 삭제할 수 없다
- [ ] 본인 배송지가 아니면 삭제할 수 없다
