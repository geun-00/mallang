# 쇼핑몰 주문 시스템 도메인

## 서비스 개요

쇼핑몰은 회원이 상품을 주문하는 온라인 커머스 서비스다.
쇼핑몰의 핵심 가치는 고객이 원하는 상품을 편리하고 안전하게 구매하는 경험을 제공하는 것이다.

유사한 서비스로 쿠팡, 네이버 쇼핑이 있다.

# 도메인 모델

## [ 회원 애그리거트 ]

### 회원 (Member)
_Aggregate Root_

#### 행위 & 규칙
- 회원 생성 : `static create(CreateMemberCommand command, PasswordEncoder encoder)`
  - 생성 시 상태는 ACTIVE
  - 가입 시간(`joinedAt`) 기록
- 회원 탈퇴 : `withdraw()`
  - `ACTIVE` 상태인 회원만 탈퇴할 수 있다
  - 탈퇴 시 상태는 `WITHDRAWN`, 탈퇴 시간(`withdrawnAt`) 기록
- 주문 가능 여부 확인 : `validateOrderable()`
  - `ACTIVE` 상태인 회원만 주문할 수 있다
- 배송지 추가 : `addShippingAddress(receiver, address)`
  - `WITHDRAWN` 상태인 회원은 배송지를 추가할 수 없다
  - 첫 번째로 추가되는 배송지는 자동으로 기본 배송지가 된다
- 기본 배송지 설정 : `setDefaultShippingAddress(addressId)`
  - `WITHDRAWN` 상태인 회원은 기본 배송지를 변경할 수 없다
  - 기존 기본 배송지는 해제된다
  - 본인의 배송지만 기본으로 설정할 수 있다
- 배송지 수정 : `updateShippingAddress(addressId, receiver, address)`
  - `WITHDRAWN` 상태인 회원은 배송지를 수정할 수 없다
  - 본인의 배송지만 수정할 수 있다
- 배송지 삭제 : `removeShippingAddress(addressId)`
  - `WITHDRAWN` 상태인 회원은 배송지를 삭제할 수 없다
  - 본인의 배송지만 삭제할 수 있다

#### 속성

- `id` : MemberId
- `nickname` : Nickname (VO)
- `email` : Email (VO)
- `hashedPassword` : Password (VO)
- `status` : MemberStatus
- `joinedAt` : LocalDateTime
- `withdrawnAt` : LocalDateTime (nullable)
- `shippingAddresses` : List\<ShippingAddress\>

### 회원 상태 (MemberStatus)
_Enum_
- `ACTIVE` : 등록 완료
- `WITHDRAWN` : 탈퇴

### 배송지 (ShippingAddress)
_Entity_

#### 속성
- `id` : ShippingAddressId
- `receiver` : Receiver (VO)
- `address` : Address (VO)
- `isDefault` : boolean

### 수신자 (Receiver)
_Value Object_

#### 속성 & 규칙
- `name` : String
  - 공백이 아니어야 한다
  - 최소 1자 이상이어야 한다
- `phoneNumber` : String
  - 010-XXXX-XXXX 형식이어야 한다

### 닉네임 (Nickname)
_Value Object_

#### 속성 & 규칙
- `value` : String
  - 공백이 아니어야 한다
  - 2자 이상 20자 이하여야 한다
  - 허용되지 않는 특수문자가 포함되면 안 된다

### 이메일 (Email)
_Value Object_

#### 속성 & 규칙
- `value` : String
  - 이메일 형식(xxx@xxx.xx)이어야 한다

### 비밀번호 (Password)
_Value Object_

#### 속성 & 규칙
- `hashedValue` : String (해싱된 값)
- 정적 생성자 `Password.encode(rawValue, encoder)`
  - 최소 8자 이상 20자 이하여야 한다
  - 영문, 숫자, 특수문자를 포함해야 한다
  - 검증 통과 후 PasswordEncoder로 해싱하여 생성

### 주소 (Address)
_Value Object_

#### 속성
- zipCode : String
- mainAddress : String
- detailAddress : String
